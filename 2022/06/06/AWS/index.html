IAM

Usergroup

  • Only contain users, can’t contain other groups
  • Same user can share different usergroups

Users

  • Can be assigned to usergroup or not
  • Inherit the policies of the usergroup belongs to

Policies

  • Json document shows service and action access

Role

  • Give permission to aws services. (e.g: EC2 needs access to other aws services, use IAM role to do that)

Access to AWS services

  • Management Console
  • AWS CLI & AWS CloudShell Add cofiguration
  • AWS SDK

EC2

Security Group

  • Set of firewall rules that control the traffic for the instance, both inbound and outbound
  • Only contain allow action
  • Classic Ports
    1. 22 = SSH (Secure Shell) - log into Linux instance
    2. 22 = SFTP (Secure File Transfer Protocol) - upload files using SSH
    3. 21 = FTP (File TRnsfer Protocol) - upload files into a file share
    4. 80 = HTTP - access unsecured websites
    5. 443 = HTTPS - access secured websites
    6. 3389 = RDP (Remote Desktop Protocol) - log into Windows instance

User Data

  • Script to bootstrap the instance for the first time. (e.g: configure the server)

Public & Private IPv4 address

  • Public IP address can be be visited from the Internet
  • Private IP address can only be visited from AWS network
  • Public IP will change when restart the instance
  • Private IP will not change when restart the instance

Connect instance by SSH

Connect instance by EC2 Connect

  • What underneath is similar to SSH connection, but everything happens remotely, no need to handle the private key file
  • Still need to configure SSH inbound rules (22) to make it work

Access to other AWS services

  1. Create IAM role for the service permission
  2. Attach IAM role to the EC2 instance

Elastic Block Storage (EBS)

  • A network drive and be specific to Availablity Zone
  • Steps of configuring EBS for EC2
    1. Create Volumes in EC2
    2. Attach Volumes to EC2 instance
  • Volume Types
    1. General Purpose SSD
      • Can be boot volume
      • gp2: Size of Volume and IOPS are linked
      • gp3: Size of Volume and IOPS are independent
      • Not support EBS Multi-attch
    2. Provisioned IOPS SSD
      • Can be boot volume
      • Need more than 16,000 IOPS, great for database workloads
      • io1: (4GB - 16TB)
      • io2 Block Express (4GB - 64TB)
      • Support EBS Multi-attach
    3. Hard Disk Drives
      • Cannot be boot volume
      • Low IOPS, good for data warehouse

        EBS Snapshot

  • A backup for the EBS, used to copy the EBS into different Availability Zone

AMI (AWS Machine Images)

  • A customized image of the EC2 Instance, can create new EC2 instace from it

EFS (Elastic File System)

  • Shared through multiple Availablity Zones
  • Configuration Steps:
    1. Create EFS, with subnets for different AZ
    2. Create Security group (sg1) for EFS, attach it to the EFS network
    3. Launch an instance with specific subnet, then add the shared file system (the EFS security group sg1 will be automatically attached)

RDS

  • ability to restore data by timestamp (backup every 5 mins)
  • can’t use SSH to connect to RDS Instance?
  • storage auto scaling

RDS Read Replicas

  • replication is ASYNC, the replicas in only used for read(SELECT; NOT INSERT, UPDATE, DELETE)
  • free if replicas in same region (even if different AZ); cost if replicas in different region

RDS Multi AZ

  • replication is SYNC, the replicas are not used, just for backup
  • can be set up from RDS Read Replicas

Aurora

  • Performance optimized underneath, automatic growth
  • Automated failover for master (Aurora instance for writes)
  • Support cross region replication
  • Reader Endpointer to connect Load Balancing with the read replicas (up to 15)
  • Backtrack: restore data at any point of time without using backups
  • can’t use SSH to connect

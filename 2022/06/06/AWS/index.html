AWS

Posted on 2022-06-06 Edited on 2022-06-08

IAM

Usergroup

  • Only contain users, can’t contain other groups
  • Same user can share different usergroups

Users

  • Can be assigned to usergroup or not
  • Inherit the policies of the usergroup belongs to

Policies

  • Json document shows service and action access

Role

  • Give permission to aws services. (e.g: EC2 needs access to other aws services, use IAM role to do that)

Access to AWS services

  • Management Console
  • AWS CLI & AWS CloudShell Add cofiguration
  • AWS SDK

EC2

Security Group

  • Set of firewall rules that control the traffic for the instance, both inbound and outbound
  • Only contain allow action
  • Classic Ports
    1. 22 = SSH (Secure Shell) - log into Linux instance
    2. 22 = SFTP (Secure File Transfer Protocol) - upload files using SSH
    3. 21 = FTP (File TRnsfer Protocol) - upload files into a file share
    4. 80 = HTTP - access unsecured websites
    5. 443 = HTTPS - access secured websites
    6. 3389 = RDP (Remote Desktop Protocol) - log into Windows instance

User Data

  • Script to bootstrap the instance for the first time. (e.g: configure the server)

Public & Private IPv4 address

  • Public IP address can be be visited from the Internet
  • Private IP address can only be visited from AWS network
  • Public IP will change when restart the instance
  • Private IP will not change when restart the instance

Connect instance by SSH

Connect instance by EC2 Connect

  • What underneath is similar to SSH connection, but everything happens remotely, no need to handle the private key file
  • Still need to configure SSH inbound rules (22) to make it work

Access to other AWS services

  1. Create IAM role for the service permission
  2. Attach IAM role to the EC2 instance
